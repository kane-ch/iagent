# Financial Metrics Query

## 功能说明

根据用户要求，查询指定公司在指定时间段内的财务指标数据，包括：

1. **基础指标**：收入、成本、毛利润、营业费用、经营利润、净利润等。
2. **加工指标**：毛利率、成本率、营业费用率、经营利润率、净利率、ROE、流动比率等。
3. **同比增长率**：对基础指标和可计算的加工指标，按同一财期与上一年同期比较。

## 可用工具

优先调用工具 `query_financial_metrics`。

工具参数：

- `ticker`：股票代码，例如 `AAPL`、`PDD`、`LI`。
- `metrics`：逗号分隔的基础指标名，例如 `Revenue,CostOfRevenue,GrossProfit,OperatingIncomeLoss`。
- `start_fiscal_year`：起始财年，例如 `2022`。
- `end_fiscal_year`：结束财年，例如 `2025`。
- `fiscal_periods`：财期，逗号分隔：`FY,Q1,Q2,Q3,Q4`。
- `source_preference`：`auto`、`online`、`cache`、`local`；默认使用 `local`。

## 执行流程

1. **识别公司和股票代码**
   - 如果用户提供 ticker，直接使用。
   - 如果用户只提供公司名称，先使用可用股票查询工具获取 ticker。

2. **识别时间范围和财期**
   - 年度查询使用 `FY`。
   - 指定季度查询使用对应季度，例如 `Q3`。
   - “所有季度”使用 `Q1,Q2,Q3,Q4`。
   - 不要无必要地查询 `FY,Q1,Q2,Q3,Q4` 全部财期，以免输出噪音。

3. **拆分指标类型**
   - 基础指标：直接放入 `query_financial_metrics.metrics`。
   - 加工指标：先转换为依赖基础指标，再在工具结果上计算。
   - 同比增长率：需要同一指标的上一年同财期数据。

4. **扩展同比查询区间**
   - 如果用户要求 `2022–2025` 的同比，实际工具查询应从 `2021` 开始。
   - 最终展示仍只展示用户要求的 `2022–2025`。

5. **调用工具查询基础指标**
   - `metrics` 参数只传基础指标，不要传加工指标名。
   - 默认 `source_preference="auto"`。

6. **计算加工指标**
   - 只用同一公司、同一财年、同一财期、同一币种/单位的数据。
   - 分母为 0 或缺失时，不计算并说明原因。
   - 余额表相关指标缺少基础项时，不要估算。

7. **计算同比增长率**
   - 同比必须比较同一财期：`FY` 对上一年 `FY`，`Q3` 对上一年 `Q3`。
   - 上年同期缺失或为 0 时，不计算。

8. **输出结果**
   - 推荐表格列：`财年`、`财期`、`指标`、`值`、`同比`、`来源`。
   - 金额类保留工具返回的币种/单位。
   - 比率类用百分比展示。
   - 缺失或不可计算项要明确说明，不要编造。

## 注意事项

- 不要主动读取 SEC 原始文件；查询逻辑已经封装在工具中。
- 不要为毛利率、经营利润率、同比等简单计算创建 Python 脚本。
- 不要把加工指标名直接传给 `query_financial_metrics`；应传其依赖基础指标。
- ROE、流动比率等指标依赖资产负债表数据。若工具未返回 `StockholdersEquity`、`AssetsCurrent`、`LiabilitiesCurrent` 等基础项，应说明当前数据不足，不能可靠计算。
- 如果工具返回多个来源的数据，优先使用同一财期、同一币种且 source 明确的数据；不要混用不同币种计算比率。

## Q4 数据推断规则

- **当某一财年只有 `FY`（全年）和 `Q1`、`Q2`、`Q3` 的财报数据，但缺少 `Q4` 数据时**，可以通过以下公式推算 Q4 数据：
  ```
  Q4 = FY - Q1 - Q2 - Q3
  ```
- 此推断规则适用于所有利润表指标：收入、成本、毛利润、营业费用、经营利润、净利润等。
- 推断的 Q4 数据需明确标注为 **"推算值 (FY - Q1-Q2-Q3)"**，与实际报告值区分。
- 资产负债表指标（如资产、负债、权益）不适用此推断规则，因为它们是时点数据而非期间累计数据。
- 如果 FY、Q1、Q2、Q3 中任意一项缺失，则无法进行可靠推算，应明确说明 Q4 数据不可用。
